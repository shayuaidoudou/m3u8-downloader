# 设计说明

## 项目边界

M3U8 下载器是一个 PySide6 桌面应用。界面层负责采集输入、展示任务与日志；下载、解密和合并逻辑由下载器及工作线程执行。视觉效果不得改变任务调度、下载线程或设置数据的语义。

## 分层结构

- `main.py`：应用启动与历史导出兼容层。
- `app/main_window.py`：主窗口组合根，只初始化共享状态。
- `app/main_window_ui.py`：布局、控件创建和主题视觉接线。
- `app/main_window_queue.py`：任务创建、队列调度和进度聚合。
- `app/main_window_lifecycle.py`：窗口生命周期、托盘、菜单与日志接入。
- `app/widgets.py`：下载工作线程、任务卡片和通用表单控件。
- `app/effects.py`：工作区氛围、空状态能量核心和流光边框。
- `app/button_effects.py`：按钮扫光覆盖层，与按钮业务事件解耦。
- `app/compose_effects.py`：左侧配置栏背景的极光光雾与有机丝带动效。
- `app/*_dialog.py`：设置、请求头、消息、搜索和线路选择窗口。
- `search.py`：搜索渠道注册表与统一工厂。
- `search_nnyy.py`：努努影院 Session 初始化、HTML 解析及按 `src_site` 聚合 M3U8 线路。

主窗口通过 Mixin 组合职责。Mixin 可以访问组合根初始化的共享状态，但不应互相创建隐式副本。

## 视觉动效设计

### 原则

1. 动效只通过绘制属性和透明覆盖层实现，不改变布局尺寸和业务状态。
2. 色彩从当前主题 token 获取，主题切换后调用 `set_tokens` 同步，不硬编码固定品牌色。
3. 空状态承担主视觉；任务出现后立即隐藏并停止对应动画，避免争夺下载信息的注意力。
4. 按钮扫光采用透明子控件，原按钮仍负责点击、键盘焦点和无障碍语义。
5. 所有持续动画在控件隐藏或用户关闭“动态特效”时停止。

### 动效层级

- 背景层：技术网格、极光、星尘和扫描带，由 `AmbientWorkspace` 绘制。
- 焦点层：旋转轨道、粒子、能量核心和点击爆发，由 `EmptyStateVisual` 绘制。
- 边缘层：沿空状态卡片周边运动的流光和信号节点，由 `EnergyBorderCard` 绘制。
- 交互层：主操作按钮的周期扫光与悬停触发，由 `ButtonShineOverlay` 绘制。
- 输入层：配置栏背景使用缓慢漂移的主题色光雾与有机丝带，避开圆环、粒子和棱角装饰，由 `ComposeEnergyCard` 绘制。

### 性能与生命周期

- 使用 Qt 属性动画驱动单一 `phase`，不创建后台线程。
- 隐藏控件时停止动画；重新显示时恢复。
- 设置中的 `ui.effects_enabled` 默认为 `true`。旧设置文件缺少该字段时保持启用，兼容已有用户配置。
- 关闭动态特效后保留静态能量核心和主题样式，只停止持续重绘、计时器和辉光脉冲。

## 主题与字号

- 全局色板和圆角来自 `config.py` 的主题 token。
- 主窗口使用 `FONT_TITLE / FONT_BODY / FONT_CAPTION`。
- 弹窗使用独立的 `DIALOG_*` 层级，避免弹窗字号随主界面密度失衡。
- 主题切换统一由主窗口 `apply_theme` 完成，并将合并后的 token 传给动态绘制组件。

## 校验策略

- 所有项目 Python 源码进行 AST 解析。
- 使用 Qt offscreen 平台实例化主窗口和弹窗，检查主题切换、绘制和动画状态。
- 动效开关需验证工作区、空状态、按钮扫光与辉光同时停止并可恢复。
- 运行现有 `pytest` 测试；外部站点和真实下载流程不纳入离线 UI 回归。

## 搜索适配器安全边界

- 努努影院适配器只主动请求配置的站点同源地址；搜索结果中的详情链接必须同源，剧集接口还必须以 `/_gp/` 开头。
- 播放接口返回的外部地址只接受 HTTP(S) M3U8，并作为候选结果交给用户选择；搜索适配器本身不继续访问第三方 CDN。
- 首页 Cookie 仅保存在当前 `requests.Session` 内，不固化到源码、设置文件或日志。
- 目标站点 HTML、接口字段和第三方媒体地址均视为不可信输入。已知风险是站点结构或风控规则变化会导致解析失败，此时返回空结果而不放宽同源校验。
- 直接下载的 M3U8 地址是用户明确选择的网络目标，播放列表中的子列表、分片和密钥地址可指向其他主机。这一能力是本地下载器的功能边界，不应将该模块暴露为服务端代理；若未来转为服务，必须增加内网地址和重定向限制。
- 爱壹帆适配器中的 MD5 仅用于兼容站点既有请求签名协议，不用于密码存储、完整性保护或本地安全决策。

## 未完成任务清理与文件安全

- 每次下载使用 `tempfile.mkdtemp` 在最终输出目录中创建唯一且由该下载器实例记录的 `.part` 工作区。
- 分片和合并产物只写入任务工作区；合并成功且产物非空后，再使用同文件系统内的 `os.replace` 发布为最终文件。
- 用户直接退出时，先通知分片线程停止并终止正在运行的 FFmpeg，等待工作线程退出后再清理工作区。
- 删除边界只限于当前下载器实例保存的专属临时路径，不扫描、不递归删除用户选择的输出目录，也不删除已发布的完整视频。
- 威胁模型是防止普通退出、取消或合并中断留下大量分片和半成品，同时防止清理逻辑越界删除用户文件。
- 已知限制：进程被强制杀死、系统断电或文件被其他进程长时间占用时，退出回调无法保证完成清理；当前不会自动扫描并删除上一次崩溃遗留的目录。

## 变更历史

### 2026-07-22 - 退出时清理未完成任务

**变更内容**：下载分片与合并产物改为写入任务专属临时工作区，成功后原子发布最终文件；直接退出、手动停止或任务失败时清理未完成产物，并支持中止正在运行的 FFmpeg。

**变更理由**：避免下载、解密或合并尚未完成时退出程序，在保存目录留下 TS 分片、concat 列表或半成品视频。

**影响范围**：下载器工作区、FFmpeg 进程生命周期、任务卡片停止逻辑和主窗口退出流程。

**决策依据**：独占临时目录能够限制删除边界；同目录暂存与原子替换确保最终文件只代表已完成任务。

### 2026-07-22 - AES 自动解密阶段日志

**变更内容**：识别到 AES-128 HLS 加密时，在日志与任务状态中明确提示“识别加密”、“加载 Key”和“开始解密”；分片并发下载前预加载一次 Key。

**变更理由**：让用户能区分普通下载与自动解密流程，同时避免首批并发分片重复请求同一 Key。

**影响范围**：AES-128 下载初始化和可观测性；非加密 M3U8 的流程不变。

**决策依据**：Key 只记录加载成功与字节长度，不在日志输出 Key 内容；后续分片复用解密器缓存。

### 2026-07-22 - 区分正片剧集与特别内容

**变更内容**：努努影院线路聚合结果增加正片与特别内容计数；线路选择窗口改为展示各线路实际可用内容和正片覆盖情况。

**变更理由**：详情页中的彩蛋、纪录片等额外播放项不应与正片剧集混称为“总集数”，避免出现“39 集 / 部分 36 集”的误导性表达。

**影响范围**：努努影院线路元数据、线路选择弹窗和提取完成日志；M3U8 结果和下载逻辑不变。

**决策依据**：站点使用 `ep<number>` 作为正片剧集标识，其他 `ep_slug` 作为特别内容；所有条目仍保留并可提取。

### 2026-07-22 - 集成努努影院搜索源

**变更内容**：新增努努影院首页 Session、关键词搜索、详情页剧集解析和 `/_gp/{影片ID}/{剧集}` 播放接口适配；多集结果按 `src_site` 聚合为可选线路。

**变更理由**：将站点专属 HTML/接口规则封装为独立搜索适配器，同时复用现有搜索结果、线路选择和 M3U8 输出流程。

**影响范围**：搜索渠道注册、搜索窗口结果展示与提取流程；下载器核心和任务调度接口不变。

**决策依据**：不固化临时 Cookie，使用同一 `requests.Session` 先访问首页再完成后续请求；详情链接和站内接口执行同源校验，外部 M3U8 只作为结果返回。

### 2026-07-22 - 配置栏动效收敛

**变更内容**：移除左侧配置栏的棱镜切面、斜向光带和边缘跑光，改为在背景中缓慢漂移的极光光雾与有机丝带；顶部品牌副标题改用主题中性色。

**变更理由**：常驻的大面积装饰干扰表单阅读，金色副标题在亮色主题中也与整体靛紫色系统脱节。

**影响范围**：`ComposeEnergyCard` 的纯绘制表现与顶部副标题颜色；布局、输入行为、任务队列和主题接口不变。

**决策依据**：保留 PySide6 原生属性动画及既有 `set_tokens` / `set_motion_enabled` 接口，以柔和、连续的背景光场代替几何装饰，兼顾视觉存在感、表单可读性与主题兼容。

### 2026-07-22 - 主界面动态视觉升级

**变更内容**：新增主题感知的粒子背景、轨道能量核心、左侧数据轨道、流光边框、按钮扫光和动态特效设置。

**变更理由**：原有波纹和低对比度极光缺少清晰的视觉焦点，界面在静态截图与实际使用中都不够有生命力。

**影响范围**：主窗口视觉层、界面设置和设置文件的 `ui` 配置；下载、搜索和任务调度接口不变。

**决策依据**：采用 PySide6 原生绘制与属性动画，无新增第三方依赖；通过显式生命周期控制限制持续绘制开销。
